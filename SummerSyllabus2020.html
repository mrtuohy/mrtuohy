Welcome to Intro CS! This class is focused on teaching the principles and logic that are key to understanding how most programming languages work, and not on teaching the specific syntax of one specific programming language.
In other words, the goal of the class is to learn how to think like a software developer. 


Why is that important?

The current workplace has lots of people who work with code – and the future will have even more. The ability to think like, understand, and communicate with software developers – whether you will be a software developer yourself or not – is an incredibly important skill that only becomes more important every year. 
This class requires no previous experience programming, but it does require hard work, flexibility, and a self-starting positive attitude!

Teaching Style Of This Class


The style of the class itself is heavily based on the concept of ‘Exploratory Learning’ –unlike other classes, I’m not going to tell you how to do it. I might tell you how to do something similar, but the whole point is to figure it out for yourself. I give a lot of hints, but you will need to think and try. If you don’t, you won’t have those ‘Ohhh – I get it!’ moments, and this class is built on those moments.

To get a bit more in-depth on this (I studied Psychology, humor me) It’s ‘Growth Mindset’ vs ‘Fixed Mindset’ – and this class is a good place to learn how to have a ‘Growth Mindset’ because this class is based on trial and error. 
This is a class where you try something, and when it doesn’t work – you’ve learned that one specific approach doesn’t work. And then you’ll need to learn why it doesn’t work, and with that new knowledge, you’ll try something else. You might eventually get the answer… but… who cares? You’ll have grown with every attempt and analysis. 
You’ll get out what you put in, and there’s always something to learn. That’s a ‘Growth Mindset’ – approaching this to see what you can get out of it.
If you come into this class with a ‘Fixed Mindset’ – then you might be the type that will get frustrated and give up or immediately ask for help when the first thing you try doesn’t work. That won’t work great in this class. 
Think of it this way instead: You aren’t wrong – you’re testing something.
Any time you place code down and run it – you’re testing it – you are experimenting – you are seeking and finding the things that work and the things that don’t.
It’s like solving a puzzle.
And is every twist of a Rubic’s Cube ‘wrong’ until the last one where everything lines up? Of course not. The same goes for Computer Science and code. 
It’s puzzle solving. It’s experimentation.
